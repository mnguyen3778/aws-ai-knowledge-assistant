# Milestone 06 – CloudWatch Dashboards and Alarms

## Objective

The objective of this milestone is to move from collecting telemetry to actively monitoring the health of the application.

CloudWatch Dashboards provide centralized visibility into application health, while CloudWatch Alarms proactively notify operators when defined thresholds are exceeded.

This milestone demonstrates operational readiness and aligns with the Operational Excellence and Reliability pillars of the AWS Well-Architected Framework.

# Dashboard Metrics

The CloudWatch Dashboard was configured to monitor the application's operational health using the following metrics.

## API Gateway

- Request Count
- 4XX Errors
- 5XX Errors
- Latency

## Lambda

- Invocations
- Errors
- Duration

## AWS WAF

- Allowed Requests

These metrics provide a centralized operational view of the application.

---

# CloudWatch Alarms

Four production-style alarms were configured.

## API-Gateway-5XX-Errors

Purpose

Detect server-side API failures.

Configuration

- Metric: 5XXError
- Statistic: Sum
- Threshold: > 0
- Period: 5 minutes
- Notification: Amazon SNS

---

## API-Gateway-Latency

Purpose

Detect slow API responses.

Configuration

- Metric: Latency
- Statistic: Average
- Threshold: > 2000 ms
- Period: 5 minutes
- Notification: Amazon SNS

---

## Lambda-Errors

Purpose

Detect Lambda execution failures.

Configuration

- Metric: Errors
- Statistic: Sum
- Threshold: > 0
- Period: 5 minutes
- Notification: Amazon SNS

---

## Lambda-Duration

Purpose

Detect slow Lambda executions.

Configuration

- Metric: Duration
- Statistic: Sum
- Threshold: > 5000 ms
- Period: 5 minutes
- Notification: Amazon SNS

---

# Amazon SNS Notifications

An Amazon SNS topic was created:

```
aws-ai-knowledge-assistant-alerts
```

Email notifications were configured so CloudWatch Alarms automatically notify operators whenever an alarm enters the ALARM state.

This provides proactive operational monitoring rather than relying on manual dashboard checks.

---

# Security Considerations

CloudWatch Alarms improve operational security by detecting abnormal application behavior early.

Examples include:

- Unexpected API failures
- Lambda execution failures
- Performance degradation
- Increased latency

SNS notifications allow operators to investigate issues before they significantly impact users.

---

# Scalability

CloudWatch is a fully managed AWS service that automatically scales with application traffic.

No additional infrastructure is required as API Gateway and Lambda usage increases.

Additional alarms and metrics can be added without modifying the application architecture.

---

# Cost Considerations

CloudWatch metrics generated by AWS services are included with standard service usage.

CloudWatch Dashboards and Alarms incur minimal monthly charges.

Amazon SNS email notifications have very low cost and fit well within the AWS Free Tier for small projects.

Overall operational monitoring cost remains very low while providing significant operational visibility.
